# Plan: Add MiMo Code Data Statistics Support

## Summary

Add MiMo Code as a new client in CalcTokens. MiMo Code stores session data in `~/.local/share/mimocode/mimocode.db` (SQLite) with a format nearly identical to Kilo. This is a straightforward adaptation of the existing Kilo SQLite parser pattern.

## Step 3: Create session parser

**File**: `crates/calctokens-core/src/sessions/mimocode.rs` (new)

Adapt from `kilo.rs` with these differences:
- Query: `SELECT m.id, s.id as session_id, m.data, s.directory FROM message m JOIN session s ON m.session_id = s.id WHERE ...`
- Use `s.id` as `session_id` (not from JSON)
- Extract `project_id` from session for workspace detection via join to `project` table
- Same JSON structure: `modelID`, `providerID`, `tokens.input/output/reasoning/cache.read/cache.write`, `time.created`, `agent`/`mode`

Key structs (same as Kilo):
```rust
struct MimoMessage { model_id, provider_id, cost, tokens, time, agent, mode }
struct MimoTokens { input, output, reasoning, cache: MimoCache }
struct MimoCache { read, write }
struct MimoTime { created, completed }
```

SQL query:
```sql
SELECT m.id, m.session_id, m.data, s.directory, p.worktree
FROM message m
JOIN session s ON m.session_id = s.id
JOIN project p ON s.project_id = p.id
WHERE json_extract(m.data, '$.role') = 'assistant'
  AND json_extract(m.data, '$.tokens') IS NOT NULL
  AND json_valid(m.data)
```

Map to `UnifiedMessage` with:
- `client = "mimocode"`
- `workspace_key = normalize_workspace_key(worktree)`
- `workspace_label = workspace_label_from_key(worktree)`
- `agent = msg.agent.or(msg.mode)`
